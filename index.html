Программирование

это процесс создания инструкций, которые компьютер может выполнять для решения конкретных задач.

Основная цель программирования

автоматизировать выполнение действий с помощью компьютера, чтобы ускорить процессы, повысить их точность и снизить вероятность ошибок, возникающих при ручном выполнении.

Основные этапы работы программы:

  • Ввод данных — программа получает данные от пользователя или из внешнего источника.
  • Обработка данных — на основе полученных данных программа выполняет заданные инструкции (вычисления, сортировка, сравнение и т.д.).
  • Вывод результатов — после обработки программа отображает результат пользователю или передает его другим системам.

Виды программ

Системные программы

обеспечивают базовое функционирование компьютера и взаимодействие между аппаратными ресурсами и пользователем

Прикладные программы

созданы для выполнения конкретных задач, которые важны для пользователя

Утилиты

вспомогательные программы, которые помогают поддерживать и оптимизировать работу компьютера

Язык программирования (ЯП)

формальная система для написания программ, с помощью которой человек может задавать инструкции для выполнения компьютером

Машинный язык

низкоуровневый язык программирования, состоящий из двоичных инструкций, понятных напрямую центральному процессору (CPU) компьютера.

Например, машинный код, Assembler

Преимущества машинного языка

Высокая скорость выполнения

машинные инструкции выполняются напрямую процессором без дополнительной интерпретации или компиляции

Точное управление аппаратурой

позволяет разработчикам максимально эффективно использовать ресурсы системы

Недостатки машинного языка

Сложность написания

разработчику необходимо писать код с использованием последовательностей нулей и единиц или их абстракций

Трудоемкость отладки

обнаружение и исправление ошибок в программе на машинном языке требует много времени и внимания к деталям

Отсутствие переносимости

программы, написанные на машинном языке, как правило, специфичны для определенной архитектуры процессора

Абстракция

концепция, позволяющая скрывать сложные детали реализации и представлять только те аспекты, которые необходимы для понимания и работы с объектами или системами

Языки низкого уровня

более высокоуровневый способ взаимодействия с аппаратным обеспечением по сравнению с машинными языками, но все же позволяют контролировать ресурсы и выполнять низкоуровневые операции. Они обеспечивают некоторую степень абстракции, но всё ещё близки к машинному коду

Например, C, C++, Go, Rust

Языки высокого уровня

языки, которые значительно абстрагируют детали аппаратного обеспечения компьютера и позволяют программистам писать код, используя понятные для человека конструкции

Например, Javascript, Python, Ruby

Характеристика языков высокого уровня

Человеко-понятный синтаксис

используют конструкции, которые приближены к естественным языкам и математическим выражениям

Кросс-платформенность

как правило, не зависят от конкретной аппаратной архитектуры и могут быть выполнены на разных устройствах без значительных изменений кода

Автоматическое управление памятью

предоставляют автоматические механизмы управления памятью, такие как сборка мусора

Преимущества языков высокого уровня

Простота использования

ближе к естественному языку, их синтаксис интуитивно понятен

Быстрая разработка

наличие встроенных библиотек и абстракций значительно ускоряет процесс разработки сложных приложений

Поддержка кросс-платформенности

могут выполняться на разных операционных системах и устройствах

Автоматическое управление памятью

поддерживают автоматические механизмы управления ресурсами

Недостатки языков высокого уровня

Меньшая оптимизация

менее оптимизированы по производительности

Сложности в управлении аппаратными ресурсами

низкоуровневых системных приложений, которые требуют детального контроля над аппаратными ресурсами

Языки программирования по типу исполнения кода

Интерпритируемые

код выполняется интерпретатором, который читает и выполняет исходный код построчно или инструкцию за инструкцией, без предварительного преобразования в машинный код

Компилируемые

код преобразуется в машинный код или промежуточный код, который может быть выполнен непосредственно процессором
